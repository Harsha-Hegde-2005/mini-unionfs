# Mini-UnionFS using FUSE

A simplified Union File System implemented in userspace using FUSE (Filesystem in Userspace).  
Merges a read-only **lower directory** and a writable **upper directory** into a single unified
mount point тАФ the same mechanism used by Docker OverlayFS.

## Features

- **Layer stacking** тАФ upper directory takes precedence over lower
- **Copy-on-Write (CoW)** тАФ modifications to lower-layer files are promoted to upper
- **Whiteout** тАФ deletions create `.wh.<filename>` markers that hide lower-layer files
- **Supported operations** тАФ `getattr`, `readdir`, `open`, `read`, `write`, `create`,
  `unlink`, `mkdir`, `rmdir`, `truncate`, `utimens`

## Usage

```bash
# Create the three required directories
mkdir -p test/lower test/upper test/mnt

# Put a file in the lower layer
echo "Hello from lower" > test/lower/file1.txt

# Mount the filesystem (keep this terminal open)
./build/mini_unionfs test/lower test/upper test/mnt
```

## Testing (open a new terminal)

```bash
# 1. Verify merged view
ls test/mnt
# Expected: file1.txt

# 2. Read from lower layer through mount
cat test/mnt/file1.txt
# Expected: Hello from lower

# 3. Trigger Copy-on-Write
echo "NEW DATA" >> test/mnt/file1.txt
cat test/upper/file1.txt   # promoted to upper: Hello from lower + NEW DATA
cat test/lower/file1.txt   # unchanged:         Hello from lower

# 4. Trigger Whiteout (delete)
rm test/mnt/file1.txt
ls test/mnt                # file1.txt gone from merged view
ls test/lower              # file1.txt still in lower (untouched)
ls -a test/upper           # .wh.file1.txt marker present

# 5. Create new file
touch test/mnt/newfile.txt
ls test/upper              # newfile.txt appears in upper

# 6. mkdir and rmdir
mkdir test/mnt/newdir
ls test/upper              # newdir present
rmdir test/mnt/newdir
ls test/upper              # newdir removed
```

## Unmount

```bash
fusermount -u test/mnt
```

## How It Works

```
Write/Delete on mount  тЖТ  always goes to upper_dir
Read on mount          тЖТ  upper_dir first, then lower_dir
Delete lower file      тЖТ  creates upper_dir/.wh.<filename> (whiteout)
Modify lower file      тЖТ  Copy-on-Write: copy to upper first, then modify
```

## Limitations

- No advanced permission or ownership handling
- Basic nested directory support
- No caching optimisation
